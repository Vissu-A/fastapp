Key components and their purposes:

core:

config.py: Configuration management using Pydantic settings
events.py: Application startup/shutdown event handlers
logging.py: Centralized logging configuration using Loguru
security.py: Security utilities (JWT, password hashing)
middleware:

Contains custom middleware like CORS configuration
Can be extended with rate limiting, authentication, etc.
api:

Versioned API routes (v1, v2)
Separation of concerns for different API versions
services:

Business logic implementation
Separation from API routes for better maintainability
models and schemas:

Clear separation between database models and API schemas
Type safety and validation
logs:

Centralized logging storage
Configured with rotation and compression
